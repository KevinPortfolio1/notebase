1. 常用資料型態 (Data Types)

類型,關鍵字,常用位元組,說明
整數,int,4 bytes,最常用的整數運算。
浮點數,float / double,4 / 8 bytes,double 精度較高，為現代開發首選。
布林值,bool,1 byte,只有 true 或 false。
字元,char,1 byte,儲存單個 ASCII 字元。
字串,std::string,變動,比 C 語言的 char[] 更安全好用。

2. C++ 核心四大特性 (OOP)

特性,名稱,實際比喻,程式中的作用
Encapsulation,封裝,提款機的機殼,將資料與操作細節藏起來，只暴露介面。
Inheritance,繼承,兒子繼承父親的姓氏,讓子類別可以使用父類別的程式碼，減少重複。
Polymorphism,多型,按下開關，燈亮或冷氣開,同一個指令，在不同物件上有不同反應。
Abstraction,抽象,開車不需懂引擎原理,只關注物件「做什麼」，而不關注「怎麼做」。

3. STL 常用容器 (Containers)

容器名稱,關鍵字,特點,建議使用場景
向量,std::vector,動態陣列，尾部增刪快。,最推薦！ 絕大多數情況的首選。
列表,std::list,雙向鏈結串列，中間增刪快。,需要頻繁在中間插入或刪除元素。
映射,std::map,鍵值對 (Key-Value)，自動排序。,像字典一樣查資料（例如：學號對應姓名）。
集合,std::set,元素不重複且自動排序。,需要過濾重複項目時。

4. 記憶體管理：舊式 vs 現代

方法,關鍵字,管理方式,優缺點
傳統方式,new / delete,手動管理,靈活但容易忘記釋放，導致記憶體洩漏。
智慧指標,std::unique_ptr,自動管理 (獨佔),物件離開作用域自動銷毀，最安全高效。
智慧指標,std::shared_ptr,自動管理 (共享),允許多個指標指向同一物件，自動計數。

5. 常見運算子符號 (Operators)
分類,符號,範例
算術,"+, -, *, /, %",10 % 3 結果為 1 (取餘數)。
關係,"==, !=, >, <",判斷是否相等或大小。
邏輯,"&&, `",
串流,"<<, >>",cout << (輸出) / cin >> (輸入)。

1. 資料清洗與標準化 (Data Cleaning)

應用場景,處理方法,範例語法 / 邏輯
移除前後空白 (Trim),erase + find_first_not_of,清除使用者輸入帳號時不小心多打的空白。
統一大小寫,std::tolower / std::toupper,搜尋「Apple」時，不論輸入「apple」或「APPLE」都能找到。
敏感字過濾,s.replace,將留言中的髒話取代為 ***。
格式檢查,regex (正規表示式),檢查輸入的是否為正確的「Email」或「電話號碼」。

2. 解析複雜格式 (Parsing)

應用場景,推薦工具,實作策略
解析 CSV / 路徑,std::stringstream,"使用 getline(ss, token, '/') 拆解網址路徑。"
固定長度協議,s.substr(),在通訊協定中，前 4 碼是 ID，後 10 碼是內容。
尋找特定標籤,s.find() + s.rfind(),抓取 HTML 標籤 <a> 裡面的連結文字。

3. 高效能字串處理：std::string_view (C++17)

在實際應用中，頻繁複製字串是導致程式變慢的主因。

    傳統做法 (std::string)：每次把字串傳給函式，電腦都會「影印」一份新的，非常耗時。

    現代應用 (std::string_view)：它不複製內容，只像是一個「放大鏡」去看原有的字串。

    優勢：在處理數百萬行日誌（Log）資料時，效能可提升 5~10 倍。
	
4. 字串應用的常見陷阱表格

陷阱,後果,解決方案
用 + 拼接數萬個字串,速度極慢（因為每次都在分配記憶體）。,先用 s.reserve() 預留空間，或使用 stringstream。
索引越界,程式直接當機（Crash）。,使用 s.at(i) 而不是 s[i] 來進行安全檢查。
編碼問題 (UTF-8),中文字變成亂碼。,"現代 C++ 建議使用 u8"""" 字串或專門的編碼庫（如 ICU）。"
