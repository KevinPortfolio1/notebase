1. Python 的型態分類：不可變 vs 可變

Python 不像 Java 那樣嚴格區分「基本型態（Primitive）」與「物件（Object）」
在 Python 中，所有變數存放的都是記憶體位址（指標），差別在於該物件是否可以被修改

不可變型態 (Immutable) — 類似其他語言的基本型態

當你修改這些變數時，Python 實際上是創造了一個新的物件，並讓變數指向它。

    代表： int, float, str, bool, tuple

    行為： 就像一個只能換掉不能修改的盒子。

可變型態 (Mutable) — 真正的引用型態

當你修改這些變數時，是在原地修改記憶體裡的內容。

    代表： list, dict, set

    行為： 就像一個可以隨時往裡面增減東西的倉庫。
	
----------------------------------------------------------------------------
語言,記憶體需求,優點,缺點
C / C++,極低 (4 Bytes),速度極快，精確控制記憶體。,容易溢位，需手動管理。
Java,中 (基本型態 4 Bytes),兼顧效能與安全。,區分基本型態與物件，邏輯較複雜。
Python,高 (28+ Bytes),絕對安全，代碼簡潔，支援無限大整數。,運算效能較低，記憶體開銷大。

資料型態,內容範例,大小 (Bytes),備註
int,0,24,數字越大，佔用空間會自動增加。
int,1,28,每增加一個位元組級別，通常跳 4 Bytes。
float,3.14,24,浮點數大小通常固定。
bool,True / False,24 / 28,其實是 int 的子類別。
NoneType,None,16,唯一的單例物件。
str,'' (空字串),49 - 51,包含編碼資訊，每增加一字元約增 1-4 Bytes。

list,[] (空),56,預留空間機制，增加元素時會成倍擴張。
tuple,() (空),40,因為不可變，比 list 更省空間。

1. 不可變型態 (Immutable) — 穩定與識別
int / float (整數與浮點數)

    計數器與索引：用於迴圈計數、清單索引。

    精確計算：Python 的 int 支援無限位數，適合處理大數據或加密演算法（如 RSA）。

    物理模擬：float 用於座標、速度、機率計算。

str (字串)

    資料傳輸：JSON 格式、API 回傳結果解析。

    日誌記錄 (Logging)：儲存系統狀態訊息。

    關鍵字識別：作為字典的 Key。

tuple (元組)

    函數回傳多值：例如 return x, y 實際上回傳一個 Tuple。

    不可變的配置：儲存資料庫連線資訊 (host, port, user)，防止在執行中意外被修改。

    座標點：(x, y) 座標，因為座標點通常是一組不可拆分的資訊。

2. 可變型態 (Mutable) — 靈活與效率
list (清單)

    有序集合：儲存使用者上傳的照片列表、購物車商品。

    堆疊 (Stack)：利用 append() 和 pop() 實現後進先出（LIFO）的邏輯。

    暫存緩衝：處理批次資料（Batch processing）時先存入 List 再統一寫入資料庫。

dict (字典/雜湊表)

    快速查找：儲存使用者 ID 與個人資料的映射（Mapping）。dict 的查詢速度是 O(1)，幾乎不受資料量大小影響。

    結構化資料：模擬實體物件（如 {'name': 'Gemini', 'age': 2}）。

    快取 (Cache)：儲存昂貴運算的結果，下次輸入相同時直接回傳。

set (集合)

    去除重複：從 10 萬筆會員信箱中取出「不重複」的名單。

    集合運算：

        交集 (Intersection)：找出兩個社群帳號的「共同好友」。

        差集 (Difference)：找出「已購買 A 商品但尚未購買 B 商品」的用戶。
