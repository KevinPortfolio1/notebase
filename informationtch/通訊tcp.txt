第一階段：建立核心觀念 (新手村)

在寫程式之前，你必須理解 TCP 的生命週期。
所有的 TCP 程式都是圍繞著「三向交握」與「連線狀態」展開的。

    Server (伺服器)：像是一間開著門的店，必須先 Listen（監聽），等客人上門。

    Client (客戶端)：像是顧客，主動發起 Connect（連線）。

    Socket (通訊端點)：想像成插座，兩端插上電纜（TCP 連線）後才能通電（傳輸資料）。

第二階段：處理多個客人的技術 (進階應用)

新手寫的簡單 Server 一次只能服務「一個」客人。
實際應用中，你需要處理成千上萬的連線，這有三種主流做法：

1. 多執行緒 / 多程序 (Multi-threading)

    做法：每當有一個新連線，就開一個新的 Thread 去處理它。

    優點：邏輯簡單。

    缺點：連線數一多（如超過 1000 個），記憶體會被耗盡。

    適用語言：Java, Python, C++。

2. 非同步與事件驅動 (Async / Event-driven)

    做法：不等待資料傳輸，而是註冊一個「當資料到時通知我」的事件。

    優點：單執行緒就能處理極大量連線。

    適用語言：Node.js (最強項), Python (asyncio), Go。

3. Go 語言的協程 (Goroutines)

    做法：利用極輕量化的「虛擬執行緒」。

    優點：寫起來像多執行緒一樣簡單，效能卻跟非同步一樣強。

    實際應用首選：目前雲端服務與後端中繼站多採用 Go。

第三階段：實際開發會遇到的坑 (實戰眉角)

當你開始寫實際專案（如：線上遊戲、即時聊天室）時，一定會遇到以下挑戰：

1. 黏包與拆包 (Sticky Packets)

TCP 是「流（Stream）」協議，沒有邊界。如果你傳送了兩個訊息：
"Hello" 和 "World"，接收端可能會收到 "HelloWorld"。

    解決方案：定義自己的「應用層協議」。例如：

        固定長度：規定每個訊息固定 1024 bytes。

        長度前綴 (推薦)：訊息開頭先放 4 bytes 代表後面資料長度。

        分隔符：訊息結尾加上 \n 或 \0。

2. 心跳機制 (Heartbeat)

有時候網路斷了（例如拔掉網線），TCP 不一定會立刻報錯。

    做法：每隔 30 秒 Server 跟 Client 互傳一個微小的封包，
	若沒回應就認定斷線並釋放資源。

3. 負載平衡 (Load Balancing)

當一台 Server 撐不住時，
會使用 L4 Load Balancer (如 HAProxy 或 Nginx) 將 TCP 流量分發到多台伺服器。

方案一：高效能聊天室 (基礎實力證明)

這不僅是傳訊息，重點在於**「狀態管理」**。

    技術要點：

        並發處理：展示你如何同時處理 100 個連線
		（使用 Go 的 Goroutine 或 Node.js 的 Event Loop）。

        使用者管理：建立一個 Map 或 Dictionary 儲存連線，實現「私訊」與「廣播」功能。

        優雅退出：當 Client 異常斷線（如強制關閉視窗），
		Server 能否正確清除該連線而不崩潰？

    面試亮點：展示你如何設計「心跳檢測（Heartbeat）」來判斷死掉的連線。

方案二：自定義協定檔案傳輸工具 (中階實力證明)

這能證明你解決了 TCP 最經典的**「黏包與拆包」**問題。

    技術要點：

        Header 設計：自定義封包頭部，例如前 4 bytes 代表檔名長度，
		後 8 bytes 代表檔案大小。

        二進位處理：不只是傳文字，而是傳送圖片或 PDF，這涉及 Byte Array 的操作。

        傳輸進度：實作 Client 端可以顯示 [====>    ] 50%。

    面試亮點：解釋你如何定義「通訊協議（Protocol）」，這證明你具備系統設計思維。

方案三：簡單的 HTTP Server (高階實力證明)

HTTP 是建立在 TCP 之上的。
如果你能用原始的 TCP Socket 刻出一個簡易的 HTTP Server，這會非常驚艷。

    技術要點：

        解析文本：手動解析 GET /index.html HTTP/1.1 這種字串。

        靜態檔案讀取：根據請求路徑，從硬碟讀取檔案並回傳。

        狀態碼實作：實作 200 OK, 404 Not Found 等基本邏輯。

    面試亮點：展現你對網路底層原理（OSI 模型）的深刻理解。
