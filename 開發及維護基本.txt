開發階段 (Development)

這是將設計稿轉化為可執行軟體的過程，強調的是品質與協作

    撰寫程式碼： 
	遵循乾淨程式碼 (Clean Code) 原則，讓他人也能讀懂

    版本控制： 
	使用 Git 工具（如 GitHub/GitLab）記錄每次更動，防止多人開發時產生的衝突

    測試 (Testing)： 

		單元測試： 確保每個小功能運作正常
        整合測試： 確保不同模組接在一起時不會爆炸

    文件撰寫： 記錄 API 規格與安裝流程，這是為了「未來的自己」與隊友著想
	
維護階段 (Maintenance)

	軟體上線（Deployment）後，真正的挑戰才開始
	維護通常佔據軟體生命週期 60% 以上 的成本

    除錯 (Bug Fixing)： 修復使用者在實際環境中遇到的異常

    效能優化： 隨著資料量變大，優化資料庫查詢速度或減少記憶體占用

    安全性更新： 針對新的資安漏洞進行補丁修補

    功能擴充： 根據市場回饋增加新功能，這會再次回到「設計」階段，形成循環
	
DevOps 的三大支柱

1. 文化與心態 (Culture)

DevOps 不僅僅是工具，更多的是文化轉變。
開發者不再只管寫程式，運維者也不再只管開伺服器；
雙方共同為產品的穩定性與成功負責。

2. 持續整合與持續部署 (CI/CD)

這是 DevOps 的技術核心。

    CI (Continuous Integration)： 
	開發者頻繁地將程式碼合併到主線，並觸發自動化測試，早期發現錯誤。

    CD (Continuous Deployment/Delivery)： 
	自動化地將測試通過的程式碼部署到環境中，讓更新能快速觸及使用者。

3. 自動化 (Automation)

凡是能自動化的過程就不手動，
包括基礎設施管理（Infrastructure as Code, IaC）、測試、部署與安全檢查。
